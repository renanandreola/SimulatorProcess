Para iniciar, você sabe o que é um processo?

Os processos representam tarefas em execução, mas nem todas têm relação direta com algum aplicativo. Muitas delas são executadas em segundo plano. Os processos CPU-Bound (orientados a CPU), são os que utilizam muito processador, logo o tempo de execução é definido pelos ciclos de processador. Os processos I/O-Bound (orientados a E/S), são os que realizam muito E/S, logo o tempo de execução é definido pela duração das operações de E/S.

Processo é uma entidade abstrata (uma estrutura no sistema operacional) que representa um software em execução. O sistema operacional atribui um identificador a cada processo, bem como descritores de arquivos, espaço de memória, prioridade, entre outras informações. Um processo pode possuir mais de um fluxo de execução (thread).


O que é um gerenciador?

O Gerenciador de Tarefas é uma das ferramentas nativas do Windows, sistema operacional da Microsoft. Com o recurso é possível visualizar todos os programas que estão abertos e rodando em segundo plano no PC, fechar aplicativos que travaram de forma rápida e ainda controlar o desempenho do seu computador. No Windows 10, o acesso é rápido pela barra de tarefas, ou do modo clássico, apertando as teclas 'CTRL' + 'ALT' + 'DEL'. Resumidamente, o gerenciador de tarefas é uma interface de controle onde o usuário detecta quais são os programas sendo executados, podendo finalizar a execução dos mesmos e controlando sua máquina.

Você sabe o que é CPU?

CPU é a sigla para Central Process Unit, ou Unidade Central de Processamento. Na CPU é onde se executam a aritmética básica, lógica, e a entrada e saída de dados, onde são realizadas as instruções para um programa rodar corretamente.

Na CPU também existem registradores, ou seja, uma memória de baixa capacidade e super velocidade. Dentro dela existem instruções, ou valores em registros, que serão usadas no processamento de cada comando. Dentro da CPU também se localiza a Unidade de Controle (UC): responsável por extrair dados da memória, decodifica-los e executa-los, consultado a ULA quando necessários. Com isso, é possível definir a CPU como o cérebro do computador.





O que são os Private Bytes?

Os Private Bytes basicamente se referem a quantidade de memória que um processo executável solicitou, porém não necessariamente à quantidade que está realmente usando, ou seja, é o tamanho atual, em bytes, de memória que o processo necessitou alocar e não pode ser compartilhado com outros processos. O nome Private, ou seja, privados, são chamados assim porque eles excluem arquivos mapeados da memória.


Você sabe o que é Working Set?

Working Set de um processo é o conjunto de páginas referenciadas por ele durante determinado intervalo de tempo, ou, segundo Denning, é o conjunto de páginas constantemente referenciadas pelo processo, devendo permanecer na memória principal para que execute de forma eficiente, evitando a elevada taxa de paginação (thrashing).

Podemos perceber que quando um programa inicia sua execução, uma elevada taxa de page faults, que são páginas que não se encontram na memória, que se estabilizam com o decorrer de sua execução, assim se a memória livre do computador estiver acima de um determinado limite, as páginas serão deixadas no conjunto de trabalho de um processo, mesmo que não estejam em uso.


O que é PID?

PID tem o significado de Process IDentification. O PID é o número de identificação de determinado processo e através dele é que temos a possibilidade de matar um processo, por exemplo. Sempre que um programa é iniciado duas vezes, como por exemplo abrir o programa desejado mais de uma vez, acaba gerando um novo PID para cada processo, mesmo que eles sejam os mesmos programas, o PID será diferente. O PID se torna importante já que é possível resolver erros na máquina, como lentidão por exemplo, muito mais facilmente, já que se torna possível ver ou determinar quando um proceso foi desovado.

Você sabe o que é Término de processos?

Quando um processo é iniciado, ele não fica em execução para sempre, em um momento ele precsia terminar. Na maior parte dos casos os processos finalizam porque ja realizaram o seu trabalho na máquina. Quando um compilador acaba de compilar e executar um programa, ele executa também uma chamada de sistema indicando para o sistema operacional que ele terminou, assim o processo finaliza.

Um outro meio de encerrar um processo, é quando usuário informa ao sistema operacional que quer cancelar um processo, no Unix, essa chamada é a Kill, já no Win32 é a TerminateProcess. Além disso, o próprio programa pode sinalizar ao sistema operacional a correção de algum erro, assim o SO interrompe o processo. Com isso, normalmente um processo finaliza pelas seguintes razões: saída normal (voluntária), saída por erro (voluntária), erro fatal (involuntário) e cancelamento por um outro processo (involuntário).


Time Sharing e Porcessamento em Lote

Time Sharring é uma técnica onde vários programas ocupam a mesma CPU, controlado por um escalonador, podendo ser definido como tempo compartilhado de programas. Time Sharring é importante porque isso permite que várias aplicações fiquem abertas ao mesmo tempo. A principal diferença de Time Sharing para processamento em lote é a utilização do computador por mais de um software ou usuário simultaneamente, já que em processamento em lote não tem como ser simultâneo.




Prioridade de Processos

A prioridade de um determinado processo, geralmente é determinada pelo sistema operacional, mas essa prioridade pode ser manipulada pelo usuário. A prioridade de cada processo afeta a quantidade de recursos disponibilizados a ele agilizando ou atrasando seus resultados e interações com o usuário.
